Root Cause

왜 긴 회의는 실패했나?

⏱️

서버 60초 한도

무료 서버는 한 번에 최대 60초만 돌 수 있음. 그 안에 끝나야 함

🤖

AI 한 번의 호출 시간

70분 오디오를 AI에게 한 번에 주면 응답 자체가 60초를 훌쩍 넘김

짧을 땐 몰랐던 벽. 실전에서만 드러나는 문제였다.

Journey

4번의 시도 끝에

1
"AI에게 시간 구간 나눠서 요청하면?"
실패Gemini가 오디오 시간 옵션을 조용히 무시. 문서엔 "영상 전용"이라 적혀 있었다 — 가정을 먼저 검증했어야 했다
2
"AI한테 '깊이 생각하지 말라'고 해야 했다"
부분 해결최신 AI는 기본적으로 "사고 모드"가 켜져 있어 답변 예산을 다 써버림. 단순 전사엔 꺼야 정상 작동
3
"모델마다 장단점이 다르다"
부분 해결Flash(빠르지만 긴 입력에 약함) + Pro(정확하지만 느림) — 단계별로 다른 모델 사용
4
"결국 녹음 자체를 5분씩 나눴다"
근본 해결처리 단계에서 쪼개려 애쓰지 말고, 녹음할 때부터 5분 단위로 저장 → 각 조각이 60초 안에 처리됨
AI 도구는 "한 번에 완벽"이 아니라 "반복 수정"이 기본. 한계도 대화하며 발견한다.
Quality Issues

동작은 했는데, 품질이 이상했다

🔁

같은 말 반복

긴 오디오 전사 시 같은 문장을 70번 반복 출력

"그래서… / 그래서… / 그래서…"
🎯

전문 용어 오인식

게임 용어를 잘못 알아들음

"어그로" → "아그로"
👻

없는 말 지어냄 (환각)

실제 회의에 없던 결정사항이 요약에 등장

"다음 주 출시 결정" (발언 없음)
Solutions

3중 안전장치

✂️

5분 분할 녹음

→ 반복 방지

짧게 나눠 주면 AI가 반복 루프에 빠지지 않음. 처리 시간도 안정화

📖

Notion 용어집 연동

→ 용어 오인식 보정

Notion DB에 용어를 등록해두면 AI가 자동으로 정확한 표기 사용. 코드 수정 없이 추가·편집

🔍

근거 인용 표시

→ 환각 탐지 가능

각 항목마다 "전사문의 이 문장이 근거"를 함께 출력. 근거가 비어 있으면 환각 의심

Differentiator

다른 회의록 앱과 뭐가 다른가

📖

용어집 Notion 연동

프로젝트 전문 용어를 Notion DB로 관리. 체크박스로 ON/OFF

현재 18개 게임 용어 등록
📋

작성 가이드 Notion 연동

회의록 스타일 규칙도 Notion 페이지에서 관리. 코드 배포 없이 튜닝

AI가 페이지를 읽고 프롬프트에 반영
🔍

환각 탐지 표시

AI 스스로 "근거 없음" 항목을 표시. 검수자가 의심 항목만 확인

전사 원문도 Notion에 함께 첨부
Takeaway

배운 점

💬
AI와 대화하며 개발

전문 프로그래머가 아니어도 가능. "만들고 싶은 것"을 말로 설명하면 구성 요소·대안·트레이드오프를 AI가 정리해준다.

🔨
첫 버전은 실전에서 깨진다

30분에선 잘 되던 게 70분에선 부서짐. 써봐야 문제가 보이고, 그제야 진짜 구조가 드러난다.

🤝
AI의 한계도 AI에게 물어서 해결

Gemini의 오디오 처리 문제를 Claude와 대화하며 파악·우회. AI 도구 하나에 종속되기보다 서로 보완시키는 관점.
