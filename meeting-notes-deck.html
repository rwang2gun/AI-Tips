Feature 1

용어집을 Notion DB로 관리

📖

AI가 잘 모를 수 있는 전문 용어를 Notion에 등록 → 전사·요약 시 자동 반영. 코드 배포 없이 실시간 편집.

소울류 카잔 브라운 더스트 메탑 리세마라 …
현재 게임 관련 용어 18개 등록 · 테스트 영상 전사 빈도 분석으로 초기 시드 구축
Feature 2

회의록 스타일도 Notion에서 관리

📋

회의록 양식·톤·분류 규칙을 Notion 페이지로 작성 → AI가 페이지를 읽고 프롬프트에 자동 반영.

✓"논의사항" / "결정사항" 분류 규칙
✓주제 50자 이내 · 아젠다 나열 금지 같은 제약
✓이슈 발견 시 문구 조정 → 다음 회의부터 반영
코드 수정·배포 없이 Notion 페이지만 고치면 반영
Feature 3

AI 스스로 "근거 없음"을 표시

🔍

요약의 각 항목에 "근거가 된 전사 문장"을 함께 출력. 검수자는 의심 항목만 확인하면 됨.

✓ 근거 있음
"다음 달까지 기획안 초안 공유"
"초안은 다음 달까지 공유하기로 하죠"
⚠ 근거 비어있음 → 환각 의심
"다음 주 출시 결정"
(전사문에 해당 발언 없음)
전사 원문도 Notion에 파일로 첨부 → 사후 검증 가능

⏱️

서버 60초 한도

무료 서버는 한 번에 최대 60초만 돌 수 있음. 그 안에 끝나야 함

🤖

AI 한 번의 호출 시간

70분 오디오를 AI에게 한 번에 주면 응답 자체가 60초를 훌쩍 넘김

짧을 땐 몰랐던 벽. 실전에서만 드러나는 문제였다.

Journey

4번의 시도 끝에

1
"AI에게 시간 구간 나눠서 요청하면?"
실패Gemini가 오디오 시간 옵션을 조용히 무시. 문서엔 "영상 전용"이라 적혀 있었다 — 가정을 먼저 검증했어야 했다
2
"AI한테 '깊이 생각하지 말라'고 해야 했다"
부분 해결최신 AI는 기본적으로 "사고 모드"가 켜져 있어 답변 예산을 다 써버림. 단순 전사엔 꺼야 정상 작동
3
"모델마다 장단점이 다르다"
부분 해결Flash(빠르지만 긴 입력에 약함) + Pro(정확하지만 느림) — 단계별로 다른 모델 사용
4
"결국 녹음 자체를 5분씩 나눴다"
근본 해결처리 단계에서 쪼개려 애쓰지 말고, 녹음할 때부터 5분 단위로 저장 → 각 조각이 60초 안에 처리됨
AI 도구는 "한 번에 완벽"이 아니라 "반복 수정"이 기본. 한계도 대화하며 발견한다.

Review

AI 리뷰로도 못 잡는 것이 있다

🤖 AI 리뷰가 잡은 것
  • 경합 조건(race) — 완료 순서 꼬임 방지
  • 재시도 폭주 — rate-limit 초과 위험
  • 네트워크 재연결 시 지수 백오프
🙋 실제로 써보고야 발견

"요약 저장 실패" 뒤 재시도 버튼을 눌렀더니…

  • 이미 완료된 전사 4개를 전부 삭제
  • 처음부터 다시 녹음 상태로 리셋
  • AI 리뷰는 이 흐름을 보지 못함
코드를 직접 못 봐도, 실제로 한 번 써보는 것이 가장 강력한 검증. AI 리뷰는 보조.

Takeaway

공유하고 싶은 실전 팁 4가지

🧪
작게 검증부터

새 기능 쓰기 전, 1분짜리 더미 입력으로 API가 정말 되는지 먼저 확인. "문서엔 된다고 적혀 있었지만 silently 무시"되는 경우가 실제로 있다.

🔎
증상에서 멈추지 말기

"한도 올렸는데 여전히 빈 응답" 같은 신호는 1차 해석의 벽. 한 단계 더 파보면 진짜 원인(예: AI의 내부 사고 모드)이 보인다.

🙋
실제 사용이 최고의 리뷰

AI가 만든 코드를 직접 읽지 못해도 괜찮다.

한 번이라도 써보는 것이 가장 강력한 검증이고, AI 리뷰는 보조.

📌
AI에게 준 규칙도 다시 묻기

문서로 기록해도 다음 작업에서 그대로 지키는 건 아니다. 리팩터·재작업 뒤엔 "그 결정 유지됐나?"를 다시 확인.
